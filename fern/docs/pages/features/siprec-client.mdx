---
title: Call Recording using SIPREC
hidden: false
---

jambonz can integrate with external recording platforms using [SIPREC](https://datatracker.ietf.org/doc/html/draft-ietf-siprec-protocol).  In this scenario, jambonz acts as a SIPREC client and forks the audio streams to send to a configured SIPREC server.

This feature is enabled via the [config](/verbs/verbs/config) verb using the `record` property.  This property can be used to start, stop, pause or resume the recording.

```json
{
  "verb": "config",
  "record": {
    "action": "startCallRecording",
    "siprecServerURL": "sip:srs@recording.example.com"
  }
}
```

## Parameters

<ParamField path="action" type="string" required={true}>
  Specifies the recording action.  
  Allowed values: 
  - startCallRecording
  - stopCallRecording
  - pauseCallRecording
  - resumeCallRecording
</ParamField>

<ParamField path="recordingID" type="string" required={false}>
  User-supplied string to identify the recording.
</ParamField>

<ParamField path="siprecServerURL" type="string" required={false}>
  SIP URI for SIPREC server.  
  Note: Required if `action` is `"startCallRecording"`.
</ParamField>
